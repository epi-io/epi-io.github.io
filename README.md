# Ahmed Azeez | Portfolio & Blog

Welcome to the source code repository for my personal portfolio and blog website. This site is built using [Hugo](https://gohugo.io/), a fast and flexible static site generator, and is styled with a custom `data-science` theme. It serves as a central hub for my professional profile, blog posts, certifications, and web applications.

## 📁 Repository Structure

This repository follows the standard directory structure of a Hugo project. Below is a comprehensive map of the files and directories, along with their specific functions.

### Root Level Files

| File | Description |
|------|-------------|
| `config.toml` | The main configuration file for the Hugo site. It defines global settings such as the base URL, site title, theme selection, author metadata, social media links, and syntax highlighting preferences. |
| `netlify.toml` | Configuration file for Netlify deployment. It specifies the build command (`hugo`), the publish directory (`public`), and environment variables like the Hugo version to ensure consistent builds across different environments. |
| `README.md` | This file. It provides an overview of the repository, its structure, and instructions for understanding the project layout. |
| `index.html` | A redirect or placeholder file at the root level, often used for specific hosting setups or to redirect users to the generated public site. |

### Core Directories

#### 1. `content/`
This directory contains all the markdown files that make up the actual content of the website. It is organized into several sub-sections:
- **`post/`**: Contains all the blog posts, organized by date and slug (e.g., `2018-05-10-cookies/`). Each folder typically contains an `index.md` file with the post content and any associated assets.
- **`about/`**: Contains the content for the "About Me" page.
- **`apps/`**: Contains markdown files describing or linking to web applications and tools.
- **`authors/`**: Contains author profiles and metadata.
- **`certs/`**: Contains information and links related to professional certifications.
- **`contact/`**: Contains the content for the contact page.

#### 2. `static/`
This directory stores all static assets that do not require processing by Hugo. These files are copied directly to the root of the generated site (`public/`) during the build process.
- **`images/`**: Contains global images such as the site logo (`logo.jpg`) and intro graphics (`introImage-dark.png`, `introImage-light.png`).
- **`js/`**: Contains custom JavaScript files, such as `certs.js` and `supabase-config.js`.
- **`apps/`**: Contains standalone HTML applications like `community-health-toolkit.html` and `cvi-calculator.html`.
- **`rmarkdown-libs/`**: Contains libraries generated by R Markdown, such as Font Awesome web fonts.
- **Favicons**: Includes `favicon.ico`, `favicon.png`, and `apple-touch-icon.png` for browser tabs and bookmarks.

#### 3. `themes/`
This directory contains the visual styling and layout templates for the site.
- **`data-science/`**: The custom theme used for this portfolio.
  - **`layouts/`**: Contains HTML templates that dictate how the content is rendered. It includes templates for the homepage (`index.html`), single pages (`_default/single.html`), lists (`_default/list.html`), and partials (reusable components like `header.html`, `footer.html`, and `social-icons.html`).
  - **`static/`**: Contains theme-specific static assets, including CSS stylesheets (`style.css`, `variables.css`, syntax highlighting themes), JavaScript files (`theme.js`, `masonry-custom.js`), and SVG icons.
  - **`archetypes/`**: Contains `default.md`, which serves as a template for creating new content files with pre-configured front matter.
  - **`theme.toml`**: Metadata about the theme itself.

#### 4. `layouts/`
This directory at the root level is used to override theme layouts or define custom shortcodes.
- **`shortcodes/`**: Contains custom Hugo shortcodes that can be used within markdown files to render complex HTML elements or embed external content easily.

#### 5. `public/`
This is the output directory where Hugo generates the final static website. **Note:** This directory is typically generated automatically during the build process (`hugo` command) and contains the compiled HTML, CSS, JS, and assets ready to be deployed to a web server or hosting platform like GitHub Pages or Netlify.

## 🚀 Deployment

This site is configured for seamless deployment. The presence of `netlify.toml` indicates it is optimized for Netlify, but it can also be easily hosted on GitHub Pages. 

To build the site locally, ensure you have [Hugo](https://gohugo.io/installation/) installed, navigate to the repository root, and run:
```bash
hugo server -D
```
This will start a local development server, typically accessible at `http://localhost:1313/`.

To generate the static files for production, simply run:
```bash
hugo
```
The generated files will be placed in the `public/` directory.
